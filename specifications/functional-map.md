# Functional Map — Albident

Карта функциональности и навигации сайта. Описывает структуру страниц, переходы между ними и языковые версии. Детальная композиция блоков на каждой странице — в `feature-specs/feature-spec-*.md`.

---

## 1. Карта страниц

| # | Страница | URL | Назначение |
|---|----------|-----|------------|
| 1 | Главная | `/` | Точка входа. Знакомство с клиникой, основные услуги, призыв к записи. |
| 2 | Услуги | `/services` | Обзорная страница со всеми услугами в виде карточек (с диапазонами цен). |
| 3 | Страница услуги | `/services/[slug]` | Динамическая. Один шаблон, отдельная страница на каждую услугу. Подробное описание, цены, FAQ. |
| 4 | О нас | `/about` | Клиника + гарантии + раздел «Команда» (5 врачей) + раздел «Отзывы». |
| 5 | Контакты | `/contacts` | Адрес, телефон, email, часы работы. |
| 6 | Запись | `/booking` | Форма онлайн-записи — главное целевое действие. |
| 7 | Политика конфиденциальности | `/privacy` | Заглушка для будущего юридического наполнения. Доступна только из подвала. |

## 2. Главное меню (header)

- **Логотип** (слева, ведёт на главную)
- **Меню:** Главная / Услуги / О нас / Контакты
- **Языковой переключатель** (TR / EN / RU)
- **Кнопка-CTA «Записаться»** (справа, выделенная, ведёт на `/booking`)

**Поведение:** header **sticky** — остаётся видимым при прокрутке страницы. Это обеспечивает постоянный доступ к навигации и кнопке записи.

**Поведение на мобильном (<768px):**

- Лого (слева).
- Компактный языковой переключатель («RU ▾», ~50 px).
- Гамбургер (справа) — открывает выезжающую панель (shadcn/ui `Sheet`).

CTA «Записаться» переезжает внутрь гамбургера, где **визуально выделен** (заливка акцентным цветом, увеличенный размер, расположен сверху панели, до пунктов меню).

Логика: на мобильном CTA доступен из гамбургера в один дополнительный тап. При этом на каждой странице уже есть собственный CTA в Hero или конце страницы + плавающая кнопка чата. Этого достаточно, чтобы освободить header от четвёртой зоны и сохранить язык-переключатель видимым (критично для дентал-туристов — см. `global-spec.md §3`, `user-stories.md` истории дентал-туриста про язык сайта).

## 2.1 CTA «Записаться» внутри страниц

Sticky header содержит постоянную кнопку «Записаться» — она доступна всегда. Внутрисайтовые CTA различаются по форме и месту:

**CTA в Hero страницы (кнопка под заголовком/описанием Hero):**

- **Есть:** `/` (главная) — точка активной конверсии в маркетинговом потоке.
- **Нет:** `/services/[slug]`, `/services`, `/about`, `/contacts`. На странице услуги кнопка переносится в конец описания (см. ниже).

**Финальный CTA-блок** — отдельный визуальный блок с заголовком вроде «Готовы записаться?» на акцентном фоне (тёмная подложка, центрированная компоновка) → кнопка ведёт на `/booking`:

- **Есть:** `/` (главная), `/services` (обзор), `/about`, `/contacts`, `/privacy` — все информационные и обзорные страницы.
- **Нет:** `/services/[slug]` — там вместо финального блока используется большая кнопка в конце read-flow (см. ниже). И `/booking` — это сама точка действия, повторный CTA не имеет смысла.

**Кнопка «Записаться» в конце read-flow (большая кнопка после основного контента, без акцентного фона):**

- **Есть:** только на `/services/[slug]` — после блока FAQ, в общем потоке текста. Это естественное завершение чтения: пациент дочитал про услугу и видит действие на месте, без визуального разрыва на отдельный акцентный блок.
- **Нет:** на остальных страницах. На них используется финальный CTA-блок (см. выше).

Общий принцип: внутрисайтовых CTA не больше, чем нужно для конверсии. Финальный CTA-блок — стандартное завершение информационных и обзорных страниц. Страница услуги — исключение: на ней та же функция выполнена через большую кнопку в общем потоке текста, чтобы не дублировать визуально акцентный блок после ёмкого read-flow «описание → этапы → цены → FAQ».

## 3. Подвал (footer)

- Контакты: адрес, телефон, email
- Часы работы
- Меню (повтор основных пунктов: Главная / Услуги / О нас / Контакты / Запись). «Запись» — обычный пункт меню без визуального выделения; акцентный CTA остаётся в header, плавающей кнопке чата и финальном CTA-блоке информационных/обзорных страниц.
- Соцсети: **Instagram**, **WhatsApp**, **Telegram**
- Ссылка на **Политику конфиденциальности** (`/privacy`)
- Копирайт

## 4. Языковые версии

Сайт доступен на трёх языках: **турецкий (TR)**, **английский (EN)**, **русский (RU)**.

**URL-схема:** локаль в пути (path-based).

- Русский (по умолчанию): `/ru/...`
- Турецкий: `/tr/...`
- Английский: `/en/...`

Примеры: `/ru/services`, `/en/about`, `/tr/booking`.

**Язык по умолчанию:** русский.

> В дальнейших спеках префикс локали в URL-примерах опускается для краткости (`/booking`, `/services/[slug]`). В коде он всегда сохраняется через `next-intl`.

## 5. Динамические маршруты

- `/services/[slug]` — страница отдельной услуги. Один шаблон, рендерится для каждой услуги по её `slug`. Полный список slug'ов — `technical-specs/technical-spec.md §3`.

## 6. Чат-ассистент

Плавающая кнопка в правом нижнем углу всех страниц, кроме мобильной версии `/booking` (там форма не должна перекрываться). Подробнее — `feature-specs/feature-spec-chat.md`.
