# Technical Spec — Чат-ассистент

Техническая реализация чата. Продуктовое описание (поведение, лимиты, состояния UI, системный промпт) — в `feature-spec-chat.md`.

Связанные документы: `feature-spec-chat.md`, общий `technical-spec.md`.

---

## 1. Поток запроса

1. Пациент пишет сообщение в плавающую панель чата.
2. Браузер отправляет историю переписки + новое сообщение в API-роут `/api/chat`.
3. Роут проверяет лимиты через `@vercel/kv` (per-IP, общий потолок). При превышении возвращает HTTP 429.
4. Роут собирает системный промпт (`lib/chat/prompt.ts`) + контекст клиники из `content/` (`lib/chat/context.ts`).
5. Роут вызывает OpenAI API (модель `gpt-4o-mini`) со стримингом.
6. Браузер получает поток токенов и постепенно отображает ответ.

## 2. Хранение переписки

Переписка живёт **только в `sessionStorage` браузера**. В БД ничего не пишется. При закрытии вкладки/окна переписка пропадает.

## 3. Лимиты

Конкретные числа (10/сессия, 20/IP в сутки, 300/сайт в сутки, `max_tokens: 300`) — в `feature-spec-chat.md §8`.

Технически:

- Счётчик per-session — в `sessionStorage` браузера.
- Счётчики per-IP и общий потолок — в Vercel KV (ключи с TTL 24 ч).
- При превышении любого из них роут возвращает HTTP 429.

## 4. Модель и параметры

- Провайдер: OpenAI.
- Модель: `gpt-4o-mini`.
- Стриминг: включён.
- Температура: `0.3` (низкая, для предсказуемых ответов в рамках границ).

## 5. Кеширование промпта

Системный промпт + контент клиники — статичные части запроса. OpenAI применяет автоматическую кеш-скидку 50% на повторяющиеся префиксы. Никакой явной настройки не требуется.
