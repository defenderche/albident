# Feature Spec — Чат-ассистент

ИИ-ассистент клиники в виде плавающей кнопки на сайте. Отвечает на вопросы о клинике, услугах, ценах, врачах. Не ставит диагнозы и не назначает лечение. При запросе на запись — ведёт на форму `/booking` с предзаполненной услугой.

Техническая реализация (поток API-роута, OpenAI, лимиты, кеширование) — в `../technical-specs/technical-spec-chat.md`.

Глобальные элементы (header, footer) описаны в `../functional-map.md` и здесь не дублируются.

---

## 1. Размещение

**Плавающая кнопка** в правом нижнем углу страницы.

- На десктопе: 60×60 px, отступ 24 px от края.
- На мобильном: 48×48 px, отступ 16 px от края.
- **Скрыта на мобильной версии `/booking`** — чтобы не перекрывать форму записи.
- Иконка — стандартная иконка чата из `lucide-react`.

При клике на кнопку открывается **панель чата** — на десктопе ~400×600 px справа снизу, на мобильном раскрывается на весь экран.

## 2. Способности ассистента

| Способность | Включена |
|---|---|
| Отвечает на FAQ о клинике (услуги, цены, контакты, часы, врачи) | ✓ |
| Удерживает медицинские границы (не ставит диагнозы, не назначает лечение) | ✓ (мандатор) |
| Кнопка-ссылка на `/booking` с предзаполненной услугой в конце ответа | ✓ |
| Помнит контекст в пределах сессии браузера | ✓ (`sessionStorage`) |
| Помогает выбрать услугу по симптомам | — |
| Сам собирает данные для записи | — |
| Сам создаёт заявку (tool-call) | — |
| Хранение переписки в БД | — |

## 3. Поведение при открытии

- Чат **не открывается автоматически**. Пациент должен сам кликнуть на кнопку.
- При первом открытии бот выводит **приветствие** (фиксированный текст, не генерируется моделью):
  - RU: «Здравствуйте! Я помощник клиники Albident. Спросите про услуги, цены, врачей — постараюсь ответить».
  - EN: «Hello! I'm the Albident clinic assistant. Ask about services, prices, or our doctors — I'll do my best to help.»
  - TR: «Merhaba! Ben Albident kliniğinin asistanıyım. Hizmetler, fiyatlar veya doktorlarımız hakkında soru sorabilirsiniz — elimden geleni yapacağım.»
- Под полем ввода — **дисклеймер**:
  - RU: «Это ИИ-ассистент. Он отвечает на вопросы о клинике, но не заменяет консультацию врача».
  - EN: «This is an AI assistant. It answers questions about the clinic, but does not replace a doctor's consultation.»
  - TR: «Bu bir yapay zeka asistanıdır. Klinik hakkındaki soruları yanıtlar, ancak doktor konsültasyonunun yerini tutmaz.»

## 4. Поведение при закрытии

- При нажатии X панель сворачивается, переписка **сохраняется в `sessionStorage`**. При повторном открытии в той же вкладке — пациент видит свой диалог.
- При закрытии вкладки/окна браузера — переписка пропадает. В БД ничего не пишется.

## 5. Локализация

Чат работает на трёх языках сайта (RU, EN, TR). Язык определяется из текущей локали URL (`/ru/...`, `/en/...`, `/tr/...`).

В системный промпт передаётся параметр `locale`. Модель отвечает на языке локали.

## 6. Системный промпт

Базовая структура промпта в `lib/chat/prompt.ts`:

1. Идентичность: «Ты — ассистент клиники Albident в Стамбуле».
2. Что разрешено: общая информация об услугах, ценах (диапазоны из `content/services.ts`), врачах (из `content/doctors.ts`), контактах и часах работы (из `content/site.ts`), FAQ, **сроки гарантий клиники** (из `../global-spec.md §10` — например, «10 лет на работу врача по имплантации»).
3. Жёсткие запреты:
   - Не ставить диагнозы.
   - Не рекомендовать конкретное лечение.
   - Не упоминать препараты и дозировки.
   - Не давать гарантий **индивидуального результата лечения** (приживётся / не будет болеть / результат после ортодонтии). Сроки гарантий клиники (см. §6.2) — это другое, их цитировать можно.
   - Не отвлекаться на темы вне стоматологии — вежливо возвращаться к делу.
   - На вопросы про точную цену — отвечать диапазоном «от — до» и фразой «точная сумма после осмотра».
4. Поведение при «остром» вопросе (боль, кровотечение, опухоль) — общая F-логика: «это требует осмотра врачом, могу помочь записаться». Без отдельных триггеров на номер скорой и подобное.
5. Использование контекста клиники, инжектится из `content/`.
6. Текущая локаль для ответа.

## 7. Кнопка-ссылка на форму записи

Когда модель в ответе упоминает конкретную услугу (имплантация, виниры и т.п.), она добавляет в конец ответа специальный маркер вида `[BOOK:implants]`. Фронтенд парсит маркер и рендерит под ответом кнопку «Записаться на имплантологию», ведущую на `/booking?service=implants`.

Список валидных слугов — в `../technical-specs/technical-spec.md §3` (раздел «Динамические маршруты»).

**Валидация на фронте.** Если модель выдаст маркер с несуществующим slug (например, `[BOOK:veneers]` — нет такой услуги), фронт **не рендерит кнопку**: он сравнивает slug с захардкоженным списком из `../technical-specs/technical-spec.md §3` и при несовпадении просто игнорирует маркер. Сам маркер из текста удаляется в любом случае, чтобы пациент не видел `[BOOK:...]` в чате.

## 8. Лимиты

- **10 сообщений на сессию** (счётчик в `sessionStorage`). При превышении — бот вежливо говорит лимит-сообщение (см. ниже) и блокирует ввод.
- **20 сообщений с одного IP в день** (Vercel KV, ключ с TTL 24 ч). При превышении — HTTP 429, фронт показывает то же сообщение.
- **300 сообщений в сутки общий потолок сайта** (Vercel KV). При превышении — то же сообщение.
- **Длина ответа модели** — 300 токенов (`max_tokens: 300`).

**Лимит-сообщение** (по языку текущей локали, попадает в `messages/{ru,en,tr}.json`):

- **RU:** «Похоже, вы много общаетесь — это здорово! Чтобы продолжить, запишитесь напрямую через форму».
- **EN:** «Looks like you have a lot of questions — that's great! To continue, please book an appointment through the form.»
- **TR:** «Anlaşılan çok sorunuz var — bu harika! Devam etmek için lütfen formdan randevu alın.»

## 9. Состояния UI

- **Idle:** панель открыта, поле ввода активно, индикатора нет.
- **Sending:** пациент нажал «Отправить», поле блокируется до начала ответа модели.
- **Streaming:** ответ модели появляется по словам в окне переписки.
- **Limit reached:** поле ввода заблокировано, под ним — сообщение про лимит и кнопка «Записаться через форму».
- **Поведение при недоступности Vercel KV.** Если KV не отвечает (ошибка или timeout запроса к счётчикам) — чат **продолжает работать** (fail-open): лимиты per-IP и общесайтовый просто не проверяются на этот запрос. Сессионный лимит 10 сообщений (`sessionStorage`) остаётся первым барьером. Обоснование: учебный проект, реального риска спама нет, fail-closed ухудшил бы UX без причины.
- **Error (сеть / OpenAI / timeout):** под последним сообщением появляется текст «Не удалось получить ответ. Попробуйте ещё раз». Кнопка «Повторить». Триггеры: HTTP 5xx от `/api/chat`, сетевая ошибка `fetch`, превышение **30 секунд** ожидания первого токена ответа модели (см. `../technical-specs/technical-spec-chat.md §4`). Если стрим оборвался уже после первых токенов — пациент видит то, что успело прийти, плюс ту же ошибку под сообщением. Автоматического retry нет — только по клику.

## 10. Доступность (a11y)

- Кнопка чата имеет `aria-label="Открыть чат с ассистентом"`.
- Панель чата — диалог с `role="dialog"`, фокус управляется при открытии/закрытии.
- Сообщения — `role="log"` с `aria-live="polite"` (новые сообщения объявляются screen-reader'ом).
- Закрытие по `Esc`.

## 11. SEO

Чат — клиентская фича, на SEO не влияет. Отдельных metadata нет.
