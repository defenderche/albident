# Feature Spec — Страница услуги (`/services/[slug]`)

Динамическая страница с подробным описанием отдельной услуги. Один шаблон, отдельный URL для каждой из 8 услуг (терапия, хирургия, имплантация, протезирование, ортодонтия, эстетика, гигиена, пародонтология). Контент в блоках различается в зависимости от услуги, структура одинакова для всех.

---

## 1. Структура страницы

Страница состоит из 7 блоков, расположенных сверху вниз:

### 1.1 Hero

- Название услуги.
- Краткое описание (1–2 строки).
- Кнопка-CTA «Записаться».

### 1.2 Подробное описание

Текстовый блок: что это за процедура, как она проводится, для кого подходит, в каких случаях рекомендуется.

### 1.3 Этапы лечения

Пошаговый процесс прохождения услуги — от первичной диагностики до завершения. Особенно важен для длительных процедур (имплантация, ортодонтия), где пациенту нужно понимать последовательность визитов.

### 1.4 Цены

Таблица подуслуг с ценами. Цена указывается как диапазон «от $X — до $Y» (USD, см. `../global-spec.md §9`). Точная стоимость определяется после осмотра.

Пример для имплантации:
- Одиночный имплант — $500–$1200
- All-on-4 — $5000–$8000
- All-on-6 — $7000–$10000

**Узнаваемые подпроцедуры.** Если услуга содержит подпроцедуры, которые пациенты ищут отдельно (например, «виниры» внутри «эстетики», «All-on-4» внутри «имплантации»), они должны быть **явно упомянуты** в:

- описании услуги (§1.2);
- таблице цен подуслуг (§1.4);
- FAQ (§1.6), если уместно.

Цель — закрыть два сценария: SEO (страница находится по поисковому запросу подпроцедуры) и узнаваемость для пациентов, которые знают название процедуры, но не знают, под какой категорией она у нас (например, ищут «виниры», не зная, что это часть «эстетики»).

В production, если какая-то подпроцедура станет главным конверсионным запросом — стоит вернуться к решению о выделении её в отдельный slug (см. `../global-spec.md §7`).

### 1.5 Связанные врачи

Карточки врачей из команды клиники, которые занимаются этой услугой. Используются те же карточки, что и на странице «О нас» (силуэт с инициалами + ФИО + роль + достижения).

### 1.6 FAQ

Частые вопросы по этой конкретной услуге с ответами.

### 1.7 CTA-блок

Повторная кнопка «Записаться» в конце страницы.

## 2. Точки выхода

Все CTA-кнопки на странице ведут на форму записи с предзаполненной услугой:

```
/booking?service=<slug>
```

Логика deeplink описана в `feature-spec-booking.md`.

## 3. Поведение при неизвестном slug

При переходе на URL с slug, которого нет в списке услуг (например, `/services/typo`), отображается стандартная страница 404.

## 4. SEO

Каждая страница услуги имеет уникальные мета-теги:

- **`title`** — шаблон: `{Название услуги} в Стамбуле — Albident` (пример: «Имплантация зубов в Стамбуле — Albident»).
- **`description`** — шаблон: `{Краткое описание услуги}. Цена от $X. Запись онлайн. Стоматология Albident, Şişli, Стамбул.`

Конкретный текст и цена для каждой услуги подставляются из `content/services.ts`. EN и TR — переводы при наполнении контентом.
