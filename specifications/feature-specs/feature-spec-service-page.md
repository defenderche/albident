# Feature Spec — Страница услуги (`/services/[slug]`)

Динамическая страница с подробным описанием отдельной услуги. Один шаблон, отдельный URL для каждой из 8 услуг (терапия, хирургия, имплантация, протезирование, ортодонтия, эстетика, гигиена, пародонтология). Контент в блоках различается в зависимости от услуги, структура одинакова для всех.

---

## 1. Структура страницы

Страница состоит из 2 блоков, расположенных сверху вниз: **Hero** и **Описание услуги**. Описание объединяет всю содержательную информацию в один читаемый блок, без визуального дробления на множество секций.

### 1.1 Hero

- **Изображение услуги** — занимает верхнюю часть блока, аспект-соотношение ~16:9. Иллюстрация процедуры, оборудования или интерьера клиники. **Без фотографий реальных людей** (см. `../global-spec.md §12`). На MVP — placeholder; владелец заменяет при наполнении контентом.
- **Название услуги** — заголовок страницы (`h1`).
- **Краткое описание** — 1–2 строки.
- **Упоминание врача(ей)** — одной строкой: «Услугу ведёт: {ФИО} — {роль}». Если врачей несколько — перечислены через запятую. Без отдельной карточки врача и без отдельной секции «Связанные врачи».

Кнопка записи в Hero **отсутствует** — переносится в конец описания, к естественному завершению read-flow (см. §1.2.5).

### 1.2 Описание услуги

Единый блок с подробной информацией. Начинается с лид-параграфа (полное описание услуги) и продолжается под-разделами с заголовками `h3` без визуального разделения секций (без меняющихся фонов, без бордеров между подсекциями — только воздух и внутренние отступы):

1. **Лид-параграф** — что это за процедура, как проводится, для кого, в каких случаях рекомендуется. Без под-заголовка.
2. **Этапы лечения** (`h3`) — пошаговый процесс от первичной диагностики до завершения. Особенно важен для длительных процедур (имплантация, ортодонтия), где пациенту нужно понимать последовательность визитов.
3. **Цены** (`h3`) — таблица подуслуг. Цена каждой подуслуги указывается как диапазон «от $X — до $Y» (USD, см. `../global-spec.md §9`). Под таблицей — короткий дисклеймер «Точная стоимость определяется после осмотра.»
4. **Частые вопросы** (`h3`) — вопросы по этой конкретной услуге в формате аккордеона (по умолчанию свёрнуты).
5. **Большая кнопка «Записаться»** — финальный элемент read-flow, ведёт на `/booking?service=<slug>`. На белом фоне в общем потоке, без отдельного акцентного блока и без заголовка «Готовы записаться?». Размер заметно больше, чем у второстепенных кнопок (`size="lg"` + увеличенная высота/паддинги), чтобы пациент, дочитавший до конца, сразу видел действие и не возвращался наверх.

**Узнаваемые подпроцедуры.** Если услуга содержит подпроцедуры, которые пациенты ищут отдельно (например, «виниры» внутри «эстетики», «All-on-4» внутри «имплантации»), они должны быть **явно упомянуты** в:

- лид-параграфе описания;
- таблице цен подуслуг;
- блоке «Частые вопросы», если уместно.

Цель — закрыть два сценария: SEO (страница находится по поисковому запросу подпроцедуры) и узнаваемость для пациентов, которые знают название процедуры, но не знают, под какой категорией она у нас (например, ищут «виниры», не зная, что это часть «эстетики»).

В production, если какая-то подпроцедура станет главным конверсионным запросом — стоит вернуться к решению о выделении её в отдельный slug (см. `../global-spec.md §7`).

## 2. Точки выхода

- **Кнопка «Записаться» в конце описания** (см. §1.2.5) → `/booking?service=<slug>` — форма записи с предзаполненной услугой. Логика deeplink описана в `feature-spec-booking.md`.
- **Sticky header «Записаться»** → `/booking` (без услуги) — доступен с любого места страницы.

**Отдельного «финального CTA-блока» с заголовком на акцентном фоне на странице услуги нет** (он остаётся атрибутом только главной). Кнопка в §1.2.5 — это естественное завершение чтения, не маркетинговый блок.

## 3. Поведение при неизвестном slug

При переходе на URL с slug, которого нет в списке услуг (например, `/services/typo`), отображается стандартная страница 404.

## 4. SEO

Каждая страница услуги имеет уникальные мета-теги:

- **`title`** — шаблон: `{Название услуги} в Стамбуле — Albident` (пример: «Имплантация зубов в Стамбуле — Albident»).
- **`description`** — шаблон: `{Краткое описание услуги}. Цена от $X. Запись онлайн. Стоматология Albident, Şişli, Стамбул.`

Конкретный текст и цена для каждой услуги подставляются из `content/services.ts`. EN и TR — переводы при наполнении контентом.
