# Visual Rules — Albident

Описывает визуальную идентичность сайта на уровне направления. Финальные значения (точные хексы, конкретные имена шрифтов, размеры) определяются при разработке в Tailwind config и `globals.css`.

Связанные документы: `global-spec.md` (тон), `technical-spec.md` (стек: Tailwind + shadcn).

---

## 1. Тон

«**Тёплый профессионализм**» (зафиксировано в `global-spec.md`):

- Современная клиника с человеческим лицом.
- **Не** холодная корпоративность.
- **Не** приторная семейность.

Все визуальные решения проверяются по этому критерию.

## 2. Палитра

**Направление:** тёплая нейтральная база + холодный акцент.

- **База** — тёплые нейтральные тона: тёплые off-white, мягкие серые, лёгкий бежевый оттенок. Создают общее «тёплое» ощущение страницы.
- **Акцент** — **deep navy** (тёмно-синий). Используется на кнопках, ссылках, ключевых интерактивных элементах. Создаёт сигнал доверия и профессионализма (стандартная медицинская ассоциация).

Точные хексы выбираются в коде с учётом контраста и доступности.

## 3. Типографика

**Направление:** sans-serif для UI + serif для крупных заголовков.

- **Sans-serif** — основной текст, кнопки, формы, навигация. Современно, нейтрально, отлично читается.
- **Serif** — крупные заголовки (Hero, заголовки секций). Создаёт ощущение премиум и «состоявшейся клиники».

Конкретные шрифты подбираются из Google Fonts через `next/font` (с подключением subsets `latin`, `latin-ext`, `cyrillic` — для всех 3 локалей).

## 4. Иконки

**UI-иконки** — интерфейсные элементы (стрелки, X, гамбургер, чат, info, чекбоксы и т.п.).

**Библиотека:** `lucide-react` (стандарт от shadcn/ui).

- Тонкие линии, минималистичный современный стиль.
- Согласуется с общей эстетикой «тёплый профессионализм».

Альтернативные UI-библиотеки (heroicons, react-icons) **не используются** — для согласованности.

**Бренд-логотипы** — отдельный класс (Instagram, WhatsApp, Telegram, логотип клиники). Это трейдмарки, а не «иконки» в UI-смысле, и `lucide-react` сознательно не гарантирует наличие всех бренд-логотипов (ради юридической безопасности библиотеки).

- Хранятся как SVG в `public/icons/`.
- Берутся с официальных пресс-страниц соответствующих сервисов.
- Инлайнятся в компонентах (footer, header, при необходимости).

Это правило независимо от текущего состава `lucide-react`: даже если какие-то бренд-логотипы там присутствуют, для согласованности и долгосрочной устойчивости все бренд-логотипы берутся из `public/icons/`.

## 5. Углы и тени

- **Углы скруглены** — радиус в районе 8–12px (`rounded-md` / `rounded-lg` в терминах Tailwind).
- **Тени мягкие** — лёгкие, не глубокие. Создают ощущение «приподнятости» элементов без агрессии.

Прямые углы и плоский дизайн **не используются** — это противоречит «тёплому» направлению.

## 6. Контраст

Все цветовые комбинации (текст на фоне, иконки, состояния кнопок) должны соответствовать **WCAG AA**:

- Минимум **4.5:1** для обычного текста.
- Минимум **3:1** для крупного текста (от 18pt / 24px) и значимых интерфейсных элементов.

Контраст проверяется при выборе финальных хексов на этапе разработки. shadcn-defaults обычно соответствуют требованиям, но при изменении основной палитры (тёплая база + deep navy) проверка обязательна.

## 7. Где живут финальные значения

- **Цвета** — CSS-переменные в `app/globals.css` (формат shadcn). Меняются в одном месте → весь сайт перекрашивается.
- **Шрифты** — подключаются через `next/font` в корневом layout, регистрируются как Tailwind-классы.
- **Радиусы и тени** — Tailwind theme (`tailwind.config.ts`), используются shadcn-компонентами.

Любая визуальная корректировка делается в этих файлах без правок отдельных компонентов.
