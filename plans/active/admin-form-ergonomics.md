# План: Эргономика админ-формы

## Цель
Сгладить шероховатости админки для владельца: компактная форма, аккуратное
удаление, ясность по витрине главной, обратная связь после сохранения.

## Контекст
- Внутренний инструмент (`/admin`), не публичный сайт — design-map не нужен.
- Файлы: `components/admin/ServiceForm.tsx`, `app/admin/DeleteServiceButton.tsx`,
  `app/admin/page.tsx`.

## Шаги
- [ ] Свернуть необязательные блоки формы (этапы / под-процедуры / FAQ) в
      раскрываемые секции со счётчиком; на edit с контентом — раскрыты
- [ ] Удаление без браузерного `confirm()` — инлайн-подтверждение «Удалить? Да / Отмена»
- [ ] Счётчик «на главной N/5» в списке услуг + подсказка про потолок
- [ ] «Сохранено ✓» — баннер на `/admin` после сохранения (через `?saved=1`)
- [ ] Проверки: `tsc`/`lint`/`test`/`build`

## Не входит
- Поиск в выпадающем списке услуг на форме записи (актуально при 20+ услугах).

## Критерии готовности
- Форма не «простыня»: необязательные списки свёрнуты.
- Удаление — без браузерного попапа; видно «N/5» на главной; после сохранения — баннер.
- Проверки зелёные.
