# План: Рисованные аватары врачей вместо силуэтов

## Цель
Заменить силуэты в карточках врачей («Команда» на `/about`, а также в блоке
«Связанные врачи» на страницах услуг) на рисованные AI-аватары. Сохранить
fallback на силуэт, если у врача нет аватара.

## Контекст
**Почему меняем принцип (след решения).** Изначально `global-spec.md §12`
запрещал изображения врачей и предписывал силуэты с формулировкой «фотографии
реальных людей не используются (учебный проект)». Пользователь решил добавить
рисованные (мультяшные) AI-аватары. Это **не нарушает** дух ограничения:
иллюстрация ≠ фото реального человека, чужие лица за наших врачей не выдаются.
Меняется только реализация: силуэт → стилизованный рисованный аватар. Запрет на
фото реальных людей сохраняется. Согласовано с пользователем 2026-06-09.

Это расхождение со спекой (вердикт «б») → спеки правим ДО кода.
Изменение вида → после спек идём через design-map, затем код.

Решения:
- Аватар + fallback-силуэт (силуэт остаётся для врачей без аватара).
- В `global-spec.md §12` — короткая сноска с причиной пересмотра (видна прямо в спеке).
- Аватары генерирует пользователь в Gemini (5 шт., единый стиль, 1:1). Я не генерирую изображения.
