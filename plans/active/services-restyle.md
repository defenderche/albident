# План: Услуги в стиле «клиника-студия»

## Цель
Распространить новый язык дизайна (открытые полосы, кикеры, ритм, де-боксинг)
на `/services` и `/services/[slug]` — первая группа внутренних страниц.

## Контекст
Язык установлен на главной (home-redesign). Skill `albident-design-map`:
сначала прототип в карте → согласование → код. Учесть находки аудита по услугам
(приоритет флагманов, навигация между услугами, таймлайн этапов вместо коробочек).

## Шаги
- [ ] 1. Дизайн-карта: прототип `/services` — открытый page-hero (кикер + h1 + лид),
      сетка 8 услуг (выделенные карточки), флагманы заметнее.
- [ ] 2. Дизайн-карта: прототип `/services/[slug]` — hero-полоса, «этапы» таймлайном,
      чистая таблица цен, строка врача, FAQ открытым списком, финальный CTA-полоса.
- [ ] 3. Показать карту, согласовать (цикл правок).
- [ ] 4. После «да» — код: ServicesHero, ServicesGrid/ServiceCard, ServiceHero,
      ServiceDetails + страницы.
- [ ] 5. Проверка: tsc, lint, test, build; визуальная сверка с картой.

## Критерии готовности
- `/services` и страница услуги читаются в едином стиле с главной.
- Карта и сайт совпадают.
- Зелёные lint / tsc / test.
