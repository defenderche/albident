# План: Правила тестов, CI, ревью PR и Supabase + README

## Цель

Закрыть несостыковки в правилах CLAUDE.md, зафиксировать решения по тестам / CI / ревью / БД, добавить README — чтобы старт скаффолда Next.js шёл без размытых мест и зависших вопросов.

## Контекст

По итогам аудита найдены:

- **Несостыковки** в формулировках Plans workflow / GitHub workflow («одним коммитом», судьба мета-задач, обязательность обновления roadmap, поведение при отсутствии plan-файла).
- **Пробелы** — нет правил по тестам, CI, ревью PR, работе с Supabase.
- **Отсутствует README.md** в корне — единственный «человеческий» документ репозитория.

Решения по 4 вопросам согласованы с пользователем:

1. **Тесты** — Vitest для критичных мест (валидация форм, лимиты чата, чистые утилиты). Без E2E, без UI-тестов.
2. **Автопроверки** — GitHub Actions на каждом PR: lint + typecheck + tests. Без pre-commit hooks.
3. **Ревью PR** — пользователь сам смотрит diff и мерджит. `/ultrareview` опционально, по запросу для крупных PR.
4. **Supabase** — миграции в `supabase/migrations/`, RLS обязателен на всех таблицах, типы генерируются в `lib/db/types.ts`.

Сам файл `.github/workflows/ci.yml` будет создан после скаффолда Next.js — сейчас фиксируем только правило в CLAUDE.md.
